⚠️ 문제점

1. boolean 플래그 + if 게이트

  • if _is_evading: return 으로 공격 차단
  • if _is_attacking: return 으로 회피 차단
  • 두 타이머가 독립적으로 누적됨
  • Attack 도중 Evade 타이머는 계속 흐름 (순서 깨짐)

2. HIT가 "상태"가 아님

  • 콜백(ticket 번호)으로 처리
  • 히트 후 공격 재시작 로직이 꼬임 (line 683-687)
  • 애니메이션 전환 시점이 불명확

3. Attack과 Evade의 상호작용

  • 회피 중에도 Attack 타이머 누적 → 회피 끝나자마자 바로 공격
  • 자연스러운 "공격 후 경직 → 회피" 흐름이 없음
  • 하나의 _process()에서 두 update()를 병렬 호출
