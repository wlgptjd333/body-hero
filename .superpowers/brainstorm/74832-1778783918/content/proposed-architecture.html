제안: 최소 FSM + Centralized Transition

"Predictable transitions" — 거대한 프레임워크 말고, 작고 단단하게

핵심 구조

enum EnemyState {
  IDLE,
  ATTACK,
  EVADE,
  HIT,
  DEAD
}

# 모든 전환은 한 곳에서
func transition_to(next: EnemyState) -> void:
    _exit_state(_current_state)
    _current_state = next
    _enter_state(_current_state)

# _process가 현재 상태 하나만 본다
func _process(delta: float) -> void:
    match _current_state:
        IDLE:   _update_idle(delta)
        ATTACK: _update_attack(delta)
        EVADE:  _update_evade(delta)
        HIT:    _update_hit(delta)
        DEAD:   return

# timer ownership centralized
# attack_timer, evade_timer는
# 각 state가 enter/exit 시에만 관리

기존 대비 변경점

✅ 유지되는 것

  • 모든 기존 난이도 스케일링 로직
  • Boss phase 시스템 (phase_changed 시그널)
  • StageConfig 기반 스탯
  • CombatDirector의 hit/miss 판정
  • 애니메이션 리소스 (AnimatedSprite2D)
  • 회피/공격 타이밍 값들 (delay min/max, duration 등)

🔄 바뀌는 것

  • boolean flags → enum state (1 current state)
  • 병렬 _update_attack + _update_evade → match state
  • ticket 방식 HIT → 상태 진입/퇴장
  • ATTACK: 단일 상태 → STARTUP/ACTIVE/RECOVERY sub-phase
  • 타이머: 각 상태 소유 → IDLE에서만 누적
