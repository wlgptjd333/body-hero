# 시퀀스 길이(seq_len) 비교 결과

**현재 학습·게임 모두 기본 시퀀스 길이는 4프레임입니다.** (`train_pose_classifier_seq.py` 인자 없이 실행 → `pose_classifier_seq_len4.keras` 학습. `udp_send_webcam_ml.py`, `pose_server.py`, `test_pose_live.py` 모두 같은 파일을 우선 로드.) 아래 비교는 참고용이며, 정확도·반응 속도 선택 시 4/12/16 실험 결과입니다.

동일 데이터·동일 하이퍼파라미터로 seq_len 4, 12, 16을 각각 학습한 검증 세트 결과 비교입니다.

## 요약 표

| seq_len | 검증 정확도 | 검증 샘플 수 | 학습 시퀀스 수 | 비고 |
|---------|-------------|--------------|----------------|------|
| **4**   | **95.7%**   | 2,534        | 22,143 → 12,668 (균형 후) | 반응 가장 빠름, none recall 낮음(87.1%) |
| **12**  | **97.5%**   | 2,056        | 16,847 → 10,108 (균형 후) | **가장 높은 정확도** |
| **16**  | **97.1%**   | 1,822        | 12,951 → 7,771 (균형 후)  | 12와 비슷, 검증 샘플 적음 |

- 검증 샘플 수가 다른 이유: seq_len이 클수록 한 녹화에서 만들 수 있는 슬라이딩 윈도우 시퀀스가 줄어들어, 균형 조정 후 검증으로 쓰이는 샘플 수도 달라짐.
- 학습 시퀀스 수: seq_len이 작을수록 같은 데이터에서 더 많은 시퀀스를 만들 수 있음.

## 동작별 recall (검증)

| 동작    | seq_len=4 | seq_len=12 | seq_len=16 |
|---------|-----------|------------|------------|
| none    | 87.1%     | **92.3%**  | 91.7%      |
| guard   | 99.0%     | **100%**   | **100%**   |
| punch_l (실험 당시 클래스명 `jab_l`) | 99.3%     | **100%**   | 97.1%      |
| punch_r (실험 당시 `jab_r`) | 98.7%     | 98.8%      | **100%**   |
| upper_l | **99.7%** | 99.6%      | 98.8%      |
| upper_r | 99.2%     | **100%**   | **100%**   |

당시 별도 클래스였던 `hook_l`/`hook_r` 검증 recall은 seq_len=4/12/16에서 각각 약 **100%** / **100%**·**98.7%** / **100%** 수준이었음. 게임·UDP·학습 라벨은 이후 **좌우 펀치 `punch_l`/`punch_r`로 통합**됨.

- **seq_len=4**: none(대기/일반) 구간을 다른 동작으로 오인하는 비율이 상대적으로 높음 (recall 87.1%). 짧은 구간만 보다 보니 노이즈·중간 자세에 더 민감한 편.
- **seq_len=12**: 전체 정확도와 none recall 모두 가장 좋고, 동작별로도 고르게 높음.
- **seq_len=16**: 12와 비슷한 수준이지만 검증 샘플이 줄어든 상태에서 97.1%. 실시간 사용 시 16프레임 버퍼로 인한 지연이 12보다 큼.

## 결론 및 권장

- **현재 학습·게임 기본**: **seq_len=4** (`train_pose_classifier_seq.py` 인자 없이 실행 시, 서버·클라이언트도 동일 파일 우선). 반응 속도가 가장 빠르며, 8프레임 대비 약간의 정확도(특히 none recall)를 양보한 트레이드오프.
- **검증 정확도**: 이 실험에서는 seq_len=12가 97.5%로 가장 높음. 정확도 우선이면 `--seq-len 12`로 학습 후 서버·클라이언트의 모델 우선순위 분기를 12 모델 파일을 보도록 맞추면 됨.
- **체감 반응 속도**: seq_len이 작을수록 버퍼가 짧아져 반응이 빨라짐. 4 > 8 > 12 > 16 순.
- **참고**: 지연 최소화 우선이면 seq_len=4가 기본이며, none 오인을 더 줄이려면 8 또는 12로 학습 후 우선순위를 그쪽으로 옮기는 방법이 있음.
